The idea

Agents need money. Humans need oversight.

Every AI agent that touches money today faces the same dead end. Either it holds custody of your funds — and you have to trust it absolutely — or it cannot spend at all. Neither is good enough for a world where agents run on your behalf around the clock.

Qoin is the third path. The agent holds nothing. It knows your address, it knows what the wallet can afford, and when it needs to pay for something, it builds a transaction and sends you a link. You tap, Face ID confirms, the money moves. The agent never touches keys. You never type a password.

The agent proposes.
The human signs.
The wallet is one HTML file.

The wallet itself is a single HTML file. No backend, no database, no subscription. Host it on GitHub Pages, your own domain, a USB stick, or any static host. Read the source. Mirror it. Fork it. The file is interchangeable; your wallets are not.

How it works

Face ID is the key.

Your iPhone's Secure Enclave stores a passkey — a cryptographic key that only Face ID can unlock. Modern browsers expose a feature called WebAuthn PRF that lets you ask the passkey to compute HMAC-SHA256 over any 32-byte input. The output is deterministic: same face, same input, same 32 bytes — always.

Qoin uses those 32 bytes as an Ethereum private key. For each wallet you create, Qoin derives a unique input from the wallet's name. Same face plus same name equals the same wallet, forever. The private key exists in JavaScript memory only during the signing moment, and is zeroed after.

// For each wallet you create: salt = SHA-256("qoin-eth-v1:" + walletName) privateKey = passkey-PRF(salt) // requires Face ID publicKey = secp256k1(privateKey) address = keccak256(publicKey) // last 20 bytes
Multi-wallet

One face, many wallets.

Different wallet names produce different keys, which produce different addresses. One Face ID, unlimited wallets.

Name them anything you'll remember: atom, tako, main, savings, pocket. Each is a silo — fund it independently, assign it to one agent, and a compromised agent loses its own envelope, not the house.

atom → HMAC-SHA256(passkey, "qoin-eth-v1:atom") → 0xA1B2… tako → HMAC-SHA256(passkey, "qoin-eth-v1:tako") → 0xC3D4… main → HMAC-SHA256(passkey, "qoin-eth-v1:main") → 0xE5F6…
For humans

A wallet, the way it should feel.

No setup

First launch, one Face ID prompt, a passkey lives in Secure Enclave. No email, no sign-up, no recovery questions.

Name and go

Type a name, Face ID, an Ethereum address appears. Copy it, fund it from an exchange. Same name always returns the same wallet.

Receive normally

Every wallet shows its address on its detail page. Incoming transactions arrive without any action from you.

Send in one tap

Pick chain, pick asset, enter amount and recipient. Face ID. Broadcast. The wallet handles gas, nonce, and signing automatically.

Any device

Open Qoin on any iPhone signed into your Apple ID. Your iCloud Keychain syncs the passkey. Re-enter wallet names. Addresses return.

Leave when you want

Wipe device clears local storage. The on-chain wallets are always recoverable from the same face plus the same names.

For agents

The protocol is the URL.

An agent has no account, no API key, no session, no login. It builds a transaction as JSON and sends you a link. That is the entire protocol.

Opening the link on any Qoin-enabled device pops the approval screen. You see the amount, the recipient, the chain, the memo, the expected sender. Face ID signs it. The network confirms it. The agent goes back to whatever it was doing.

Proposal JSON

{ "v": 1, "chain": "base", "to": "0xRecipient…", "value": "100000", "token": "USDC", "memo": "API subscription", "from": "0xYourAtom…" }

Proposal URL

The JSON is base64url-encoded and attached to the wallet host:

https://qoin.io/wallet.html#
w=atom&tx=eyJ2IjoxLCJjaGFpbiI6…

Send it over any channel — Telegram, email, SMS, WhatsApp. The link carries no secret; interception is harmless.

What agents can and cannot do

Can: read balances and transaction history (public blockchain data), construct arbitrary transactions, send you proposal links, sign the proposal with their own key for provenance.

Cannot: sign on your behalf, move money without a human tap, create wallets, learn your private key, see your other wallets.

Security

What Qoin protects you from. And what it doesn't.

This is a wallet, not a bank. The honesty of that framing is the whole security model.

Protected

  • Phishing sites copying Qoin's HTML — passkeys are bound to origin.
  • Network interception of proposal links — the link contains no secret.
  • Compromised agents — an agent can propose but not sign.
  • Seed-phrase theft — there is no seed phrase to steal.
  • Cloud breaches — there is no cloud. Keys live only in Secure Enclave.
  • One wallet leaking to another — each wallet is siloed by derivation.

Not protected

  • Losing your iPhone and wiping iCloud. There is no recovery.
  • Someone with your face and your unlocked iPhone.
  • Proposals you approve without reading. The details are shown — read them.
  • Bugs in the HTML you loaded. Audit the file before trusting real money.
  • Malicious mirrors at look-alike domains. The domain is the wallet's identity.
You trust Apple's Secure Enclave. You trust the HTML file you loaded. You trust no agent, no server, no cloud, no person.

Features in v0.1 alpha

What works today.

Face ID setup

One-tap passkey creation, bound to the hosting origin. No passwords, no seed phrases.

Multi-wallet

Unlimited named wallets derived from the same Face ID. Each is a separate Ethereum address.

Send ETH

Native ETH on Ethereum and Base. Manual send flow with chain picker, amount, and memo.

Send stablecoins

USDC and USDT on both supported chains. Non-standard ERC-20 behavior handled correctly.

Receive anything

Every wallet shows its address. Standard Ethereum inbound — no action required from you.

Balance reading

Live balances for ETH, USDC, USDT on Ethereum and Base via public RPC. No API keys.

Agent proposal links

URL-based protocol. Agents build JSON, encode base64url, send a link. You tap and sign.

Proposal builder

Inside the wallet, create proposal URLs to share or test. Share via native iOS share sheet.

Block explorer links

Every wallet links out to Etherscan and Basescan for full transaction history.
